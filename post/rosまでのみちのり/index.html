ROSまでのみちのり

18 Dec 2017, 19:00

経緯

ROSを使って物質的なマッピングと音響的なマッピングを同時に行うことを 目的に，マイクロホンをラズパイマウスにくっつけて動かしたい． そこで上田先生の本(Amazon商品ページ)を 参考にしながら実装などを進めているが，つまずいてしまった点などまとめて今後使えるメモにしたいところ．

環境

  • Machine: Raspberry Pi3 & RaspberryPiMouse V2
  • Machine OS: Ubuntu Server 16.04 LTS
  • Host OS: Ubuntu 16.04 LTS (&El Capitan)

ROS導入およびテストで詰まったところ

  1. reboot is needed
  2. rosdep init
  3. pip(in Travis CI)

reboot is needed

症状
Fixing recursive fault but reboot is needed! と表示されて立ち上がらない
対策
必要なだけ再起動してあげる
原因
不明（わかる方いらしたら教えてください）

今でもそこそこな頻度で起こる問題． 表示された内容を日本語訳すると「再帰的な問題を治しているのだけど，再起動してほしいんだ」のような感じ．
言われた通り再起動するがこれが2,3回では済まないので何かOS入れる際に失敗したのかと思い何度も入れなおす結果に． 結局，たくさん再起動することで治るので問題ない（？）

もともと別の用途で使っていたRasbianの入ったmicroSDでROS導入しようとし，この問題にぶつかり，OSが原因なのでは？ と思い参考にした本でも使っているUbuntu Serverに変更したという情けない経緯があります． なのでいまだにOSのせいなのか不明です．

rosdep init

症状
rosdep init が実行されない
対策
学校でやろうとしない
原因
プロキシーサーバを使用するとアクセスができない

依存関係含めたインストールやアップデートなどがおこなえるrosdepコマンド． インストール後に初期化が必要なので，必ずsudo rosdep initは入力することになるかと思います． しかし，rosdep initはproxy環境下ではエラーが起こることが分かっています． 実際に実行してみると

cannot download default sources list from:  
https://raw.github.com/ros/rosdistro/master/rosdep/sources.list.d/20-default.list  
Website may be down.

と表示されます．しかしウェブブラウザでそのURLにアクセスすると正常に表示することができます． ネット上ではsudo -E rosdep init (sudoの-Eオプション)でできるという情報が転がっていましたが， 私の環境下ではうまくいきませんでした． 自宅にラズパイを持ち帰り，自宅のネットワークに繋いだ状態でsudo rosdep initを実行したところ 無事動いたので，やはりネットワークが原因だと考えられます．

pip (in Travis CI)

症状
pip install が拒否される
対策
pip install --user で実行
原因
権限など

上で述べた本を進めていくとTravis CI という継続的インテグレーションツールを用いたROSのテストを行う内容になります． そのテストを行うにあたり環境を構築する必要があり，その最中にpipコマンドを用いてインストールをする部分があります． ここで詰まってしまいましたが，吐かれたエラー文を見るとPermission deniedとあったので権限などの問題のようです． --userをつけることによってユーザディレクトリにインストールされるため回避できるようになったと思われます．

まとめ

細かいところでつまずくと次へのモチベーションが結構下がるけれども，英語のドキュメントなどを探して なんとか克服する．ただそれだけで妙に達成感あるのが危険だといつも思う． 今後も何か見つけたらここにあげたいと思います．何か訂正点や追記すべき事項などありましたらContactにお願いします．
