Hugoでサイトを立ち上げる

27 Dec 2017, 17:38
Hugo / website / Github

hugoで個人のページを作った(ここ)

Githubをきちんと使えるようになりたいという願望や個人のサイトを持ちたいという願望が合わさって，
現在のwebサイトができました．

幸運なことに同じような形で自分のページを持っている人がいたり，ネット上にもノウハウが多く転がっていたので助かりました． その中でも自分が特に詰まった点や，他のノウハウを載せているページには見つけられなかった注意点など紹介したいと思います．

もくじ

  1. gh-pages ブランチが不要
  2. 継続的にページを更新するために
  3. 別PCに編集する環境を作る

gh-pages ブランチが不要

いくつかのノウハウを載せているページを参考にしながらこのHAMPOTを立ち上げましたが， それらの多くはgh-pagesブランチを切ってそこにpushするように書かれていました． しかし今ではGithub Pagesもアップデートされ，その手続きは不要となりました． (gh-pagesブランチを使っても正常に機能するそうです．)

継続的にページを更新する

静的サイトを立ち上げられるよ！で終わる人もいれば，さくらサーバやGithub Pagesで公開できるよ！ で終わる人もいますが，それではブログになりません．

ブログでないにしても簡単に更新できるようにする必要があります．

ビルドと公開を同時に

hugoでサイトのビルドができ，publicフォルダが生成（更新）されます．このディレクトリをGithubにpushすることで，ページが更新されます． この一連の流れをシェルスクリプトにしてしまうとかなり幸せになれると思います．

#!/bin/bash
rm -rf public/post/*
hugo
cd public
git add .
git commit -m -"update pages"
git push origin master

別PCに編集する環境を作る

今回はこれがメインだったりします．
最初にMacBookProでhugoを導入してサイトを作っていたのがMBPが諸事情により使えなくなってしまったので，
Ubuntuの入ったPCで引き続き更新しようとしたところ，思いの外詰まってしまいました．

やったことは改めて環境構築(hugoのインストール)とbitbucketを用いた環境の引き継ぎです．
Ubuntuでのinstallにはapt-getせよといろんなところに書いてありますが，バージョンが古いせいかうまく動作しなかったです． （具体的にはsafeHTMLAttrが定義されていないとかでhugoコマンドによるビルドが通らない状況に陥りました．）
そのため公式ではsnapを使うように指示していますし， コマンドラインが苦手な人でもhugoのGithubページ(hugo Releases)から直接ダウンロードしてきて，これを利用する方法もあります．

GitHubに上がっているのはpublicフォルダだけなのでこれをpullしてきてもそのまま引き継ぎができません．
そこで環境そのものを引っ越しするために無料でプライベートのリポジトリが作れるbitbucketを用いました．
使い方はほとんどGithubと同じで，元の環境でpushして引き継ぎしたいPC側でcloneすればOKです．
自分はhugo new site hogehogeを先にしておいて，cloneしたディレクトリから適宜必要なものを copyする方法で引き継ぎを行いました．

まとめ

Hugo + Githubで自分のサイトを作るだけで結構いろいろなことを学べる機会になっている気がする．それと近々ハムスターに関連する記事をあげたい．
